# CODEBUDDY.md — 珑点智算 · Muse 复刻 AGENT 宪法

> 本项目专属宪法。每次开工前自动读取，必须遵守。
> 每条规则后附「🗣 大白话」——说人话解释它在管什么。

---

## 0. 项目身份

- **项目**：珑点智算 · Muse 复刻（[GitHub: chun162/muser](https://github.com/chun162/muser)）
- **任务**：一比一复刻 [TopenMuse 词元妙思](http://topenrouter.com/muse) 的功能与体验
- **定位**：本地优先、零后端的"多入口 AI 创作平台"
- **对标核心功能**：图片生成 · 视频生成 · 无限画布 · AI 工具箱 · 画风库 · BYOK 统一模型注册
- **总目标**：功能与体验一比一对齐，**品牌与资产自有化**

🗣 大白话：咱们这个项目叫"珑点智算"，要照着 TopenMuse 做一个功能体验一样好用的产品；但牌子、文案、图片都得是咱们自己的。

---

## 1. 先设计，后编码

- 未经用户明确同意进入编码阶段前，**禁止编写业务代码**（应用源码）
- 允许做的事：调研、写文档、设计架构、初始化仓库、`.gitignore`、配置模板、示意资源
- 进入编码前，先确认里程碑与范围已对齐

🗣 大白话：现在是"定方案、画图纸"的阶段。只有用户点头说"开始写代码"，才能上手；但搭环境、写方案随时可以做。

---

## 2. 复刻合规红线

- ✅ **允许**：功能形态、信息架构、交互流程、画风库分类框架的**对齐**；自有文案与自有资产
- ❌ **禁止**：复制对标源码、Logo/字体/受版权保护的预览图等受保护资产；逐字照搬其营销文案
- "一比一"以**体验对齐**为准，不以"盗用资产"为手段；不冒充对方品牌、不造成用户混淆

🗣 大白话：可以"学明白"别人家的产品并做出一样好用的版本，但不能抄代码、扒图标、盗文案。自己的牌子、自己的文案、自己的图——合规也体面。

---

## 3. 密钥与隐私

- 用户的 API Key **仅可存于浏览器本地**（IndexedDB / localStorage），绝不可写入代码、提交到 Git、打印到日志、或外发
- 仓库内不得出现任何真实密钥、token、凭证；只允许占位符与示例（如 `YOUR_API_KEY`）

🗣 大白话：用户的模型密钥是用户的私人物品，只能留在他自己的浏览器里。绝不能写进代码、存进仓库、打到日志里——那是把家门钥匙贴大街上。

---

## 4. Git 与仓库卫生

- **初始化仓库前**：先清理内部资料、密钥、本地配置；确认无敏感文件后再 `git init`
- 始终保持 `.gitignore` 覆盖：`.env*`、密钥、本地数据库、大体积资源、`node_modules`、`dist`
- 每次提交前自检：跟踪文件中不得含密钥 / 本地配置

🗣 大白话：开新仓库的第一步不是写代码，而是把"隐私垃圾"扫干净，用 `.gitignore` 这张网兜住。密钥、本地缓存、大文件——保证永远不会被提交上去。

---

## 5. 架构默认（除非用户另定）

- **前端**：React + Vite + TypeScript + Tailwind CSS
- **画布**：ReactFlow（待集成，TopenMuse 用的是 ImportMap 加载的 ReactFlow）
- **本地存储**：IndexedDB + localStorage
- **模型接入**：MVP 纯 BYOK（浏览器直连，零后端）；后续才考虑 VIP 托管算力
- **坚持"本地优先 · 零后端"** 作为核心卖点（与对标一致）

🗣 大白话：技术选型先按这套默认来。模型调用初期让用户自己填密钥直连、不搭后端；以后才考虑做付费算力。

---

## 6. 决策以文档为准

- `README.md` + `docs/` + `docs/specs/` 是项目的**唯一事实来源**（目标、范围、决策、里程碑）
- 已决策事项不得擅自推翻；若要变更，先更新文档并说明理由
- 立项阶段的结论（范围、技术栈、商业模式）具有约束力，直到被显式修订

🗣 大白话：方案以仓库里的文档为准，别凭记忆或拍脑袋改方向。已经定好的事不能偷偷推翻——改就先改文档、说清楚为什么。

---

## 7. 范围与交付节奏

- **全量目标**：四大入口（图片 / 视频 / 画布 / 工具箱）+ 画风库 + 本地优先 + BYOK + VIP 托管算力
- **分阶段执行**：
  - P1 MVP：图片生成 + 画风库 + 本地优先 + BYOK
  - P2：视频生成模块
  - P3：无限画布 + AI 工具箱
  - P4：VIP 托管算力
- **MVP 必须闭环再扩展**——不牺牲基础质量赶进度

🗣 大白话：最终四个入口全要做，但别一口气全上。先让"出图+存图+自带密钥"这条线跑通，再一步步加视频、画布、工具箱。MVP 必须是一个完整的"能用"的产品。

---

## 8. 写作风格

- **默认简体中文**
- **结论先行**——先说结果，再展开理由
- 能用表格对比就用表格，能列清单就别写大段落
- 涉及数字/配置/路径，准确给出，不要模糊

🗣 大白话：说人话、说重点、说清楚。先告诉结果，有必要再解释为什么。

---

## 9. 自主干 vs 先问

- ✅ **大胆干（本地）**：读文件、查资料、分析、写文档、跑本地命令、生成示意/占位资源
- ⚠️ **先确认**：删除/覆盖项目文件、推送远程仓库、任何对外有影响的动作

🗣 大白话：在项目本地放开手脚干；但凡是会"出家门"的动作，先打招呼。

---

## 10. 信任但核实

- 工具或子代理说"做完了"——**自己亲眼验证**再交差
- 结论必须有依据；拿不准就说"不确定"，别编造、别含糊带过

🗣 大白话：别人（或工具）说完成了，你要自己看一眼是不是真完成了。不知道就是不知道，别编。
