## 3. 核心功能拆解

### 3.1 图片生成
- **文生图**：输入提示词，一句秒出画面。
- **图生图**：以参考图为基础重绘/风格化。
- 推测能力：分辨率、步数、采样器、负面提示词、批量出图。

### 3.2 视频生成
- **文生视频 / 图生视频 / 首尾帧**：生成短视频片段。
- 首屏主推 **Seedance 2.0 满血版**（字节 Seedance 视频模型），VIP 享有高优算力、不排队。
- 推测能力：时长、帧率、运动强度、分辨率。

### 3.3 无限画布
- 节点/画布式创作空间，可自由摆放草图、参考图、生成结果，串联多步创作。
- 同类参考：tldraw / Excalidraw / Canvas 节点编排。
- 价值：把"图片 + 视频 + 参考"组织成可迭代的项目，而非孤立出图。

### 3.4 画风库（Style Library）
风格即"预设流水线"——每个风格封装了模型、提示词模板与参数。分类与样例：

| 分类 | 风格样例（原文） |
|------|------|
| 2D 动漫 | 日式动漫、经典动画、国风水墨、韩系条漫、都市言情、国风漫画、惊悚漫画、美式漫画、复古年代、儿童绘本、二次元仙侠 |
| 3D 动漫 | 皮克斯3D、赛博未来、国风3D、史诗奇幻、现代3D动漫、游戏仙侠 |
| 写实风格 | 现代写实、国风写实、生存写实、乡村写实、大唐盛世、未来写实 |
| 自定义风格 | 用户可自建风格（提示词 + 参数模板） |

> 画风库是产品的"内容护城河"：风格描述文本即轻量资产，可自有沉淀，避免依赖第三方版权素材。

### 3.5 本地优先 · 零后端
- 作品、项目、偏好全部存于**浏览器本地**（IndexedDB / localStorage）。
- 不上云、不登录 → 隐私友好、零账户摩擦、首屏即用。
- 这是区别于 Midjourney / 即梦等"云端账户型"产品的核心定位。

### 3.6 统一模型注册（Unified Model Registration）
- 用户在某处注册自己的模型 API Key（OpenAI / Stability / Replicate / 自建推理服务等）。
- 应用按注册信息**本地路由**请求，无需平台后端中转用户数据。
- 即典型的 **BYOK（Bring Your Own Key）** 模式，也是"零后端"得以成立的技术前提。

## 4. 商业模式推断

| 层级 | 体验 | 算力来源 | 是否需后端 |
|------|------|----------|-----------|
| 免费层 | 本地创作、BYOK | 用户自带密钥直连模型 | 否（纯前端） |
| VIP 层 | "不排队"、极速通道、超清出片 | 平台托管高优算力（如 Seedance 满血版） | 是（需账户 + 推理服务） |

> 注意："零后端"描述的是**默认/免费体验**；VIP 增值说明平台仍存在托管推理服务。复刻时需明确：我们是否也做托管算力，还是先纯 BYOK。

## 5. 技术架构推断（用于立项参考，非源码）

| 层 | 合理选型（推断） | 说明 |
|----|------|------|
| 前端框架 | React / Vue + Vite | SPA，纯前端即可运行 |
| 画布 | tldraw / Excalidraw / Konva / Fabric | 无限画布的可选方案 |
| 本地存储 | IndexedDB（作品/项目）+ localStorage（偏好/密钥） | 支撑"本地优先" |
| 密钥管理 | 浏览器本地加密存储，绝不出库 | 合规红线 |
| 模型路由 | 浏览器直连模型 API（或轻量 CORS 代理） | BYOK 核心 |
| 样式 | TailwindCSS 等原子化方案 | 现代 AI 工具常见 |

> 上表为**基于公开页面与行业通法的推断**，非对标的实现真相。实际选型见立项设计书 §6。

## 7. 可复刻点 vs 合规边界

| 可复刻（建议） | 不可复刻 / 需注意 |
|------|------|
| 产品形态：四入口 + 画风库 + 本地优先 + BYOK | 对方**源码、私有算法、品牌标识（Logo/字体）** |
| 信息架构与交互流程 | 逐字照搬的营销文案（改写为自有表达） |
| 风格**分类框架**与自建风格机制 | 直接抓取对方风格预览图等受版权保护素材 |
| "本地优先·零后端"的工程思路 | 冒充对方品牌或造成用户混淆 |

> 结论：**功能与体验一比一对齐，品牌与资产自有化**。这是可持续、合规的复刻路径。

---

## 8. 小结

TopenMuse 的竞争力不在单点模型，而在 **"四入口聚合 + 画风库内容 + 本地优先零后端 + BYOK"** 的组合，以及用 VIP 托管算力做变现。复刻的成败取决于：① 画风库的体量与质量；② 本地优先体验是否真的顺滑；③ BYOK 路由是否稳定；④ 是否提供有吸引力的增值算力。
