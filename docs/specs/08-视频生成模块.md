# 08 · 视频生成模块（P2）

> 实施真源 · P2 模块。依赖：画风库(02)、模型注册(04)、本地存储(03)、数据契约(06)、后端 `POST /api/video`。
> 本模块在 MVP(P1) 之后启动，流程与 01(图片生成) 对齐。

## 1. 功能

| 子功能 | 说明 |
|--------|------|
| 文生视频 (Text-to-Video) | 输入 prompt（+ 可选画风），生成短视频片段 |
| 图生视频 (Image-to-Video) | 上传参考图 + prompt，基于参考生成视频 |
| 首尾帧 (First-Last Frame) | 提供首帧与尾帧 + prompt，生成过渡视频 |

## 2. 用户输入与默认值

| 字段 | 必填 | 默认 | 备注 |
|------|------|------|------|
| prompt | 是 | — | 可结合画风模板 |
| mode | 是 | `text2video` | `text2video` / `image2video` / `firstlast` |
| 参考图(refImage) | image2video/firstlast 时 | — | 本地读取，不主动外传 |
| 首帧/尾帧 | firstlast 时 | — | 两图 |
| 画风 | 否 | 无 | 注入 prompt 模板 |
| duration | 否 | 4（秒） | 受 Provider 能力约束 |
| fps | 否 | 24 | |
| resolution | 否 | 1024x576 | |
| motion | 否 | medium | 运动强度 low/medium/high |
| batch | 否 | 1 | |

## 3. 交互流程

```
首页 → [视频生成]
  → 选模式 + 填 prompt（+ 参考图/首尾帧 + 画风 + 参数）
  → [生成] → 调后端 POST /api/video（BYOK 代理，见 04）
  → 返回视频(URL/Blob) → 预览 → [保存到作品](写 03 IndexedDB)
```

## 4. Mock 阶段

- 无真实模型时，前端用占位资源（海报图 + "视频(占位)"标注）跑通流程；后端 `POST /api/video` 代理逻辑与生成接口一致，仅端点不同。
- 真实接入后，视频以 URL 或 Blob 返回，前端用 `<video>` 预览。

## 5. 验收标准

- [ ] 三种模式（文生/图生/首尾帧）均可触发生成（mock 占位可跑通）。
- [ ] 选画风后生成实际生效（对比可见差异）。
- [ ] 参数（时长/帧率/分辨率/运动）生效。
- [ ] 结果可预览（`<video>` 或占位）并保存到本地作品（IndexedDB）。
- [ ] 后端 `POST /api/video`：校验、权限、错误信封与 `POST /api/generate` 一致；密钥不存储/不日志。

## 6. 后端接口

- `POST /api/video`：入参与 `POST /api/generate` 同构（provider/baseUrl/apiKey/model/prompt/params），`params` 含 `duration/fps/resolution/motion`；转发到 `{baseUrl}/videos/generations`（openai-compatible 视频端点）。
- 错误映射、权限、涉及表：与 `POST /api/generate` 一致（无状态代理，无表）。
