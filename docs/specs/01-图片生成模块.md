# 01 · 图片生成模块

> 实施真源 · P1 模块。依赖：画风库(02)、模型注册(04)、本地存储(03)、数据契约(06)。

## 1. 功能

| 子功能 | 说明 |
|--------|------|
| 文生图 (Text-to-Image) | 输入 prompt，可选画风，生成图片 |
| 图生图 (Image-to-Image) | 上传参考图 + prompt，基于参考重绘/风格化 |
| 参数控制 | 分辨率、步数、采样器、负面 prompt、批量张数 |
| 结果管理 | 出图后预览、保存到"我的作品"、可基于结果再编辑 |

## 2. 用户输入与默认值

| 字段 | 必填 | 默认 | 备注 |
|------|------|------|------|
| prompt | 是 | — | 若已选画风，画风 prompt 模板自动拼接（见 02） |
| 参考图 | 图生图时必填 | — | 本地读取，不主动上传到非用户指定端点 |
| 画风 | 否 | 无（原图风） | 选中后注入 preset |
| 分辨率 | 否 | 1024×1024 | 受 Provider 能力约束 |
| 步数 steps | 否 | 30 | |
| 采样器 sampler | 否 | 模型默认 | 如 dpmpp_2m |
| 负面 prompt | 否 | 空 | |
| 批量张数 batch | 否 | 1 | 上限按 Provider 限制 |

## 3. 交互流程

```
首页 → [图片生成] 
  → 输入 prompt（+ 可选参考图 / 画风 / 参数）
  → [生成] → 调 04 路由 → 04 直连 Provider（见 06 契约）
  → 返回图片 → 预览 → [保存到作品](写 03 IndexedDB)
  → 可选 [基于此再编辑] 回到输入（带入上图作参考）
```

## 4. 验收标准

- [ ] 文生图：填 prompt 即可出图（默认参数）。
- [ ] 图生图：上传参考图 + prompt 可出图；参考图仅用于本次请求，不落库（除非用户另存）。
- [ ] 选画风后生成，实际生效（对比无画风结果可辨差异）。
- [ ] 参数调节生效；批量出图返回 N 张。
- [ ] 生成中可取消；失败有提示且可重试。
- [ ] 结果可保存到本地作品并被持久化（见 03）。

## 5. 边界与注意

- 不内置任何模型权重；全部经 04 的 BYOK 路由。
- 大图预览用对象 URL / Blob，避免不必要上传。
- 跨域：若 Provider 不支持 CORS，走 04 的代理策略（可选），但密钥仍不出本地。
