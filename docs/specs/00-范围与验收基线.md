# 00 · P1 范围与验收基线

> 实施真源 · 本批仅 P1(MVP)。编码以本文件划定的边界为准。

## 1. P1 范围（做）

| 模块 | 范围 | 关键能力 |
|------|------|----------|
| 图片生成 | 文生图 / 图生图 | prompt、参考图、参数、批量出图 |
| 画风库 | 分类 + 内置风格 + 自定义风格 | 一键应用风格（注入 prompt+参数） |
| 本地优先存储 | 作品/项目存 IndexedDB，不登录 | 刷新不丢、隐私友好 |
| 统一模型注册（BYOK） | 用户配置自有 API Key，本地路由 | 浏览器直连模型，零后端 |

## 2. 不在 P1（不做，留给 P2-P4）

- ❌ 视频生成（P2）
- ❌ 无限画布 / AI 工具箱（P3）
- ❌ VIP 托管算力 / 账户体系（P4）

## 3. 技术默认（来自 docs/03 决策，除非用户另定）

| 项 | 默认 |
|----|------|
| 前端框架 | React + Vite |
| 样式 | TailwindCSS |
| 本地存储 | IndexedDB（作品/项目/自定义风格）+ localStorage（偏好/密钥引用） |
| 画布 | P1 不需要（tldraw 留待 P3） |
| 模型接入 | 纯 BYOK，浏览器直连，零后端 |
| 语言 | TypeScript |

## 4. 验收基线（Definition of Done）

P1 视为完成，当且仅当：

- [ ] **可运行**：`npm run dev` 起前端，首屏可访问。
- [ ] **首屏到出图 ≤ 3 步**：进入图片生成 → 填 prompt（可选手选画风）→ 出图。
- [ ] **本地优先**：未登录即可使用；生成记录存 IndexedDB，刷新/重开不丢。
- [ ] **画风库可用**：内置 ≥ 20 风格，分 2D/3D/写实/自定义四类；自定义风格可增删改并持久化；一键应用能正确注入 prompt 模板与参数。
- [ ] **BYOK 可用**：用户可在设置中配置至少一个 Provider 的 Key；配置后图片生成能真实调用并返回结果。
- [ ] **密钥安全**：Key 仅存本地，Git 仓/日志/网络请求体以外均不可见；无真实密钥入库。
- [ ] **错误处理**：鉴权失败 / 跨域 / 限流 / 超时 有明确提示，不白屏。
- [ ] **合规**：无对标源码、无受保护素材、文案自有化（见 CODEBUDDY.md §2）。

## 5. 依赖与顺序建议

```
04 统一模型注册(BYOK)  ← 调用链底层
06 数据模型与接口契约  ← 全局类型/schema
03 本地优先存储        ← 依赖 06 的 schema
02 画风库              ← 依赖 06
01 图片生成            ← 依赖 02(画风) + 04(BYOK) + 03(存储) + 06
05 信息架构与路由      ← 串联以上页面
```

> 建议实现顺序：06 → 03 → 04 → 02 → 01 → 05。
