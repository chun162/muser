# 07 · 数据库设计（MVP 这一批）

> 实施真源 · 数据库设计。本文件为 MVP 这批的**存储模型唯一真源**。
> 范围：仅 MVP（图片生成 / 画风库 / 本地优先存储 / 统一模型注册 BYOK）。
> 存储形态：**浏览器本地 IndexedDB（零后端）**，非服务端 SQL。本设计**不含任何服务端库**。
> 决策依据：已与产品方确认 4 项设计点（见 §6）。本文件生成后**暂不改动 `src/lib/db.ts`、暂不建表**，落地改动见 §7。

## 1. 存储模型说明

- 物理落地 = IndexedDB **对象存储（object store）**，不是关系表；但用"实体/字段/键"方式表达领域模型，便于评审。
- 全部数据存于浏览器，不上云、不登录（呼应"本地优先·零后端"卖点）。
- IndexedDB **不支持外键/级联**，关系靠 id 引用在应用层维护（见 §5）。
- 密钥（`apiKey`）仅存本地 IndexedDB，绝不出库、不写日志、不经任何服务端（CODEBUDDY.md §3）。

## 2. 业务对象（实体）

### 2.1 Generation（生成记录 / 作品）— 核心
| 字段 | 类型 | 键 | 说明 |
|------|------|----|------|
| id | string | PK | uuid |
| prompt | string | | 用户输入提示词 |
| params | object | | { resolution, steps, sampler, negativePrompt, batch } |
| styleId | string? | FK→Style | 应用的画风 id（可空；可指向内置或自定义） |
| providerId | string? | FK→ModelProvider | 实际使用的模型 id（**本次新增**，用于追溯） |
| resultBlob | Blob | | 结果图（内联，无外链） |
| createdAt | number | 索引 | 时间戳（毫秒） |
| refImage | Blob? | | 图生图参考图；**默认不随作品保存**（见 §6-D） |

### 2.2 Style（画风）
| 字段 | 类型 | 键 | 说明 |
|------|------|----|------|

## 3. 对象存储设计（目标 schema）

| Store | keyPath | 索引 | 内容 |
|-------|---------|------|------|
| `generations` | id | createdAt, styleId, providerId | 用户作品 |
| `styles_custom` | id | category | 用户自定义画风 |
| `providers` | id | enabled | **新增**：BYOK 模型配置 |
| `settings` | key | — | 偏好（theme/lastRoute） |
| `projects` | id | updatedAt | 占位（P3 画布启用） |

> 当前 `src/lib/db.ts` 已实现 `generations` / `styles_custom` / `settings` / `projects`；`providers` 为本次设计新增，待 §7 落地。

## 4. 表关系（ER）

```
Style(1)          ──<  Generation(N)    styleId（可选）
ModelProvider(1)  ──<  Generation(N)    providerId（本次新增，可选）
Generation 内联 resultBlob（自包含，无外链）
无 User / Account（本地优先·零后端，无登录）
```

| 关系 | 基数 | 引用字段 | 级联 |
|------|------|----------|------|
| Style → Generation | 1:N | Generation.styleId | 无（删 Style 不删作品） |
| ModelProvider → Generation | 1:N | Generation.providerId | 无 |
| Settings | 独立 | — | — |

## 5. 数据完整性与查询

- **无外键**：删除自定义 Style 时，引用它的 Generation 不级联删除；读取时若 `styleId` 在合并列表中找不到（内置被改/自定义被删），UI 显示"已删除画风"，不报错。
- **按模型查询**：`generations` 建 `providerId` 索引，可按模型统计/筛选作品。
- **按画风查询**：`generations` 建 `styleId` 索引；注意内置风格不在库中，其 styleId 解析依赖运行期合并的 `styles` 列表（见 §2.2 说明）。
- **排序**：作品列表默认按 `createdAt` 倒序。

## 6. 设计决策（已确认）

| # | 决策点 | 结论 | 理由 |
|---|--------|------|------|
| A | ModelProvider 存储 | **独立 `providers` store** | 结构清晰、便于按模型查询与扩展；避免 `settings` 成大杂烩 |
| B | Generation.providerId | **新增** | 追溯"这张图用什么模型生成"，为真实 BYOK 调用做准备 |
| C | 内置 Style 入库 | **保持 seed 不入库** | 内置风格是随版本发布的静态资产，避免用户库重复存储 |
| D | refImage 随作品保存 | **默认不保存** | 参考图仅用于当次请求，省本地存储；字段保留可选 |
