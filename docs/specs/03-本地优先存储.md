# 03 · 本地优先存储

> 实施真源 · P1 基础设施。依赖：数据契约(06)。被 图片生成(01)/画风库(02) 写入。

## 1. 原则

- **本地优先 · 零后端**：所有用户数据存于浏览器，不上云、不登录即可用。
- 隐私边界：作品、项目、自定义风格、密钥引用 **不出本机**（除非用户显式调用自带 Key 直连模型）。
- 兼容性：使用 IndexedDB（结构化数据）+ localStorage（轻量偏好/密钥引用）。

## 2. IndexedDB 库设计

库名：`ciyuan-muse`（版本见 06 `DB_VERSION`）。

| Store | keyPath | 内容 | 索引 |
|-------|---------|------|------|
| `projects` | id | 项目/画布草稿（P1 仅占位结构，P3 启用） | updatedAt |
| `generations` | id | 单次生成记录：prompt/params/结果 Blob 引用/画风id/时间 | createdAt, styleId |
| `styles_custom` | id | 用户自定义风格（见 02） | category |
| `settings` | key | 偏好、UI 状态、密钥引用（见 04） | — |

> 生成结果图片以 **Blob** 存 `generations.resultBlob`（或对象 URL 映射），避免外链。

## 3. 数据模型（示意，待实现）

见 `06`：`Generation`、`Project`、`Style`、`Settings`。

## 4. 隐私与合规

- 不收集用户行为埋点（除非后续明确加且告知）。
- 提供"数据本地存储"说明入口（设置页），呼应对标卖点。
- 提供"清空本地数据"能力（危险操作，二次确认）。

## 5. 验收标准

- [ ] 未登录可完成全部 P1 流程。
- [ ] 生成作品存 IndexedDB，刷新/重开浏览器仍在。
- [ ] 自定义风格持久化（见 02）。
- [ ] 提供"清空本地数据"且需二次确认。
- [ ] 无数据向非用户指定端点外发（除 BYOK 直连模型外）。

## 6. 注意

- 配额：IndexedDB 有容量上限，大图建议压缩后存；超出时优雅提示。
- 多标签页同步非 P1 必需，但写入需防并发覆盖。
