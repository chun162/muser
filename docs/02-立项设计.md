# 02 · 立项设计书：珑点智算 · Muse 复刻

> 版本：v0.1（讨论稿，未进入编码）
> 对标：TopenMuse 词元妙思
> 范围：本阶段产出设计决策与范围草案；编码另开 Sprint

---

## 1. 项目目标

构建一款**本地优先、零后端**的多入口 AI 创作平台，与 TopenMuse 在功能与体验上**一比一对齐**，但使用 **珑点智算** 自有品牌与资产。

- 对齐项：四大创作入口、画风库、本地优先存储、统一模型注册（BYOK）。
- 差异项：品牌、视觉资产、文案自有化；模型接入与变现策略待决策（见 §7、docs/03）。

---

## 2. 目标用户与场景

| 用户 | 场景 | 核心诉求 |
|------|------|----------|
| 二次元 / 插画创作者 | 文生图、图生图、挑画风出图 | 出图快、风格多、不折腾 |
| 短视频 / 内容创作者 | 文生视频、首尾帧 | 短片段快速产出 |
| 设计师 / 概念artist | 无限画布串联多步创作 | 可迭代、可组织 |
| 隐私敏感用户 | 本地存储、不登录 | 数据不出本机 |
| 重度用户 | VIP 极速算力 | 不排队、超清 |

---

## 3. 产品定位与差异化

- **定位**：浏览器里即开即用的 AI 创作工作台，数据归用户自己。
- **一句话**：把"出图、出视频、画布、风格"装进一个不上云的工具里。
- **差异化抓手**（复刻基础上的可选增强）：① 更强的画风库运营；② 更顺的本地优先体验；③ 透明的 BYOK 成本；④ （待定）自有托管算力。

---

## 4. 功能范围

> 目标：**全量对齐对标四入口**（图片 / 视频 / 画布 / 工具箱 + 画风库 + 本地优先 + BYOK + VIP 托管算力）。
> 执行：**分阶段交付**，MVP 先跑通图片闭环，再依次扩展（见 §8）。

### MVP（P1 首版，先闭环）
- [ ] 图片生成：文生图 / 图生图
- [ ] 画风库：分类 + 内置风格（≥20）+ 自定义风格
- [ ] 本地优先存储：作品/项目存 IndexedDB，不登录
- [ ] 统一模型注册：用户配置自有 API Key，本地路由（BYOK）

### 扩展（P2–P4）
- [ ] 视频生成：文生视频 / 图生视频 / 首尾帧（P2）
- [ ] 无限画布：节点式多步创作（P3）
- [ ] AI 工具箱聚合：扩图、局部重绘、超分等（P3）
- [ ] VIP 托管算力（不排队 / 超清）——依赖后端账户体系（P4，待决策细化）

> 视频与画布复杂度高，MVP 先图片闭环验证"本地优先 + 画风库 + BYOK"核心体验，再扩展至全量对齐。

## 5. 信息架构设计（我们怎么做）

```
珑点智算 · Muse（拟）
├── 首屏
│   ├── 价值主张：本地优先 · 零后端
│   └── 四入口卡片：图片 / 视频 / 画布 / 工具箱
├── 图片生成
│   ├── 提示词 / 参考图 / 参数
│   └── 画风库选择（内嵌或跳转）
├── 视频生成（完整版）
├── 无限画布（完整版）
├── 画风库
│   ├── 分类：2D / 3D / 写实 / 自定义
│   └── 风格详情：预览 + 一键应用
├── 模型注册（设置）
│   └── 添加/管理 API Key（本地加密）
└── 我的作品（本地）
    └── IndexedDB 列表 / 项目
```

## 7. 数据 / 模型接入方案

- **方案 A（纯 BYOK，MVP）**：用户自带密钥，浏览器直连模型。零后端、零成本、隐私最佳；变现弱。
- **方案 B（BYOK + 托管算力，P4）**：免费 BYOK + VIP 托管推理（类 Seedance 满血版"不排队"）。需账户体系与推理服务，复杂度高。
- **路线（已决策）**：MVP 走方案 A 快速验证体验；P4 引入 VIP 托管算力做增值变现。

## 9. 风险与合规

- **合规**：不复制对方源码/品牌资产/版权素材；文案自有化；见调研文档 §7。
- **密钥安全**：API Key 仅存本地，严禁入库与日志。
- **模型稳定性**：BYOK 依赖第三方 API 可用性，需做好错误提示。
- **范围蔓延**：视频/画布复杂度高，务必先 MVP 闭环。

---

## 10. 成功指标（建议）

- 首屏到出图 ≤ N 步；本地项目可持久化、刷新不丢。
- 画风库风格数 ≥ 对标量级（约 20+ 内置风格）。
- （第二阶段）VIP 转化与排队时长。
